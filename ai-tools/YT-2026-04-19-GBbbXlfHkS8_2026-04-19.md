## Краткое содержание
В этом видео обсуждается новая функция "Routines" от Claude Code, позволяющая запускать автоматизированные задачи по расписанию, но имеющая значительные ограничения по количеству и частоте запусков. Автор предлагает обходное решение: создание собственного, неограниченного планировщика задач для AI-агентов (таких как Cloud Code, Gemini, Codex) с использованием набора специальных Markdown-файлов. Эти файлы позволяют обучить AI-агента выполнять рутинные операции, такие как сбор информации из YouTube, обработка заметок в Obsidian, формирование отчетов или напоминаний, полностью автономно и без внешних лимитов. Видео демонстрирует, как настроить агента, передав ему эти инструкции, и подчеркивает универсальность предложенного метода для различных AI-платформ.

## Ключевые моменты
*   **Новая функция Claude Routines:** Недавно анонсированный инструмент от Claude Code, позволяющий запускать AI-агентов для выполнения задач по расписанию, аналогично автоматическим "реагентам".
*   **Проблемы с серверами Claude:** Из-за высокой нагрузки на свои серверы, Claude активно продвигает использование Routines для выполнения регулярных задач в непиковые часы, чтобы снизить общее потребление ресурсов.
*   **Ограничения Claude Routines:** Даже на PRO-тарифе Claude Routines имеет строгие ограничения, например, до 5 запусков рутинных задач в день с минимальным интервалом в 1 час, а также требует интеграции с GitHub-репозиторием.
*   **Сравнение с N8n:** Функционал Routines от Claude напоминает возможности сервиса N8n, который автор ранее активно использовал для автоматизации и планирования задач для AI.
*   **Отсутствие памяти контекста:** Одним из ключевых недостатков Claude Routines является то, что он не запоминает предыдущий промпт, что делает его менее эффективным для сложных, многоступенчатых задач.
*   **Пример использования Routines:** Автор демонстрирует свой настроенный "рутин" для Claude Code, который собирает видео с YouTube по различным тематикам (Codex, Claude Code, Gemini, OpenCloud) в определенные дни недели.
*   **Детальный сбор информации:** Помимо самих видео, система извлекает из них краткое объяснение содержания, что позволяет быстро понять суть видео без просмотра.
*   **Система оценки и внедрения:** Полученная информация из видео проходит через систему рейтингов, определяющих потенциальное влияние на работу, и отмечается как "сделано" после внедрения.
*   **Интеграция с Telegram:** Вся релевантная информация и команды для внедрения удобно отправляются в Telegram, где их можно скопировать и использовать для своих AI-агентов, таких как Claude Code, Gemini или Codex.
*   **Основная цель видео:** Показать, как настроить своего AI-агента на выполнение схожих рутинных задач, но без использования сторонних сервисов вроде N8n и без ограничений, присущих Claude Routines.
*   **Предлагаемое решение:** Использование набора из шести Markdown-файлов, которые нужно передать своему AI-агенту. Эти файлы содержат инструкции для самостоятельного внедрения неограниченного таймера и планировщика задач.
*   **Универсальность решения:** Предоставленные файлы являются универсальными и могут быть внедрены в различных AI-агентов, включая CloudCodeBot, GeminiBot и Codex, адаптируясь под их особенности.
*   **Экономия токенов и обход лимитов:** Правильно настроенный AI-агент с этим планировщиком позволяет эффективно парсить информацию и внедрять элементы, минимизируя расход токенов и полностью обходя тарифные ограничения.
*   **Примеры автоматизируемых задач:** В файлах содержатся идеи для планировщика AI-агента, такие как автосохранение сессий, еженедельная дефрагментация памяти, группировка и чистка старых заметок.
*   **Рекомендации для предпринимателей:** AI-агент может быть настроен на утренние брифинги (сбор информации о конкурентах или предстоящих задачах), еженедельные отчеты в Telegram или мониторинг упоминаний бренда.
*   **Рекомендации для бухгалтеров/финансистов:** Бот может напоминать о сроках сдачи отчетности и уплаты налогов, автоматизируя рутинные финансовые задачи.
*   **Процесс внедрения:** Автор демонстрирует, как загрузить или передать ссылки на Markdown-файлы из Obsidian своему AI-агенту (на примере Codex) и проинструктировать его на внедрение таймера и планировщика.
*   **Использование Obsidian для хранения инструкций:** Markdown-файлы могут быть добавлены в Obsidian, и затем AI-агенту можно передавать ссылки на эти заметки, что упрощает управление инструкциями.
*   **Преимущества персонализации:** Такой подход позволяет давать AI-агенту персональные примеры задач, адаптированные под конкретные нужды пользователя, вместо шаблонных.

*   **Claude Code (Claude)**: AI-ассистент, большая языковая модель (LLM), основная платформа, вокруг которой строится обсуждение.

*   **N8n**: Инструмент для автоматизации рабочих процессов и интеграций. Упоминается как аналог Claude Routines, который автор использовал ранее.
*   **GitHub repo**: Сервис для хостинга и управления версиями кода, требуемый для некоторых функций Claude Routines.
*   **Gemini**: AI-ассистент от Google, большая языковая модель (LLM), упоминается как одна из платформ, на которой можно внедрить предложенные решения.
*   **ChatGPT Codex / Codex**: AI-ассистент, вероятно, на базе модели OpenAI (например, GPT), также поддерживающий внедрение кастомных планировщиков.
*   **Telegram**: Мессенджер, используемый для получения отчетов от AI-агента и скачивания инструкций и команд.
*   **Obsidian**: Инструмент для управления знаниями и создания связанных заметок в формате Markdown. Используется для хранения файлов-инструкций для AI-агентов и обработки заметок.
