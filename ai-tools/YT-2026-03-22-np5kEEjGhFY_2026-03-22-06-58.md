## Краткое содержание

Видео посвящено оптимизации работы AI-агентов с использованием подписки Claude Plus (17-20 долларов) вместо дорогостоящей Claude Max. Автор делится тремя основными стратегиями для экономии токенов и оптимизации работы агентов: интеграция Claude Code через Telegram для удобства непрограммистов, использование MCP-подключения к N8n для делегирования рутинных задач отдельным sub-агентам, и внедрение Obsidian как интеллектуальной системы хранения информации. Главный тезис — правильная архитектура экосистемы AI-агентов позволяет значительно снизить расходы без потери функциональности и качества результатов. Видео предназначено для людей, которые хотят эффективно использовать AI-инструменты при минимальных затратах, и для тех, кто создаёт контент на разные платформы.

## Ключевые моменты

- **Подписка Claude Plus как оптимальный выбор**: Большинство пользователей предпочитают Claude Plus (17-20 долларов) вместо Claude Max, так как хотят решать рутинные задачи с минимальными затратами, а не подписываться на премиум-версию. Автор также использует подписку Plus, чтобы понять потребности аудитории и найти способы оптимизации работы своих E-агентов.

- **Интеграция Claude Code через Telegram**: Один из способов оптимизации — подключение Claude Code через Telegram вместо работы в терминале. Это делает инструмент доступным для людей, далёких от программирования, позволяя им писать задачи прямо в удобном интерфейсе мессенджера.

- **MCP-подключение к N8n для делегирования рутинных задач**: Вместо перегрузки основного Claude Code рутинными задачами (просмотр видео, создание субтитров, оптимизация видео для YouTube), можно создать отдельного sub-агента через MCP-подключение к N8n, который будет выполнять эту работу независимо.

- **Экономия на выполнении задач через sub-агентов**: Просмотр видео и извлечение смысла стоит 50 копеек — 1 рубль за задачу. Создание субтитров с использованием трёх агентов (Claude, Gemini, ChatGPT) через API обходится в 1-2 рубля за каждого. На неделю (3 видео) такие работы обходятся примерно в 18 рублей для оптимизации видео под SEO.

- **Obsidian как интеллектуальная система хранения памяти**: Obsidian — это бесплатный аналог Evernote, хранящий заметки в Markdown-формате. Он решает проблему недоступности информации с других устройств и позволяет создавать структурированную базу знаний для AI-агентов.

- **Ограничения локального хранилища контента**: До внедрения Obsidian Cloud Code сохранял заметки в локальной папке на сервере, что делало их недоступными с других устройств. Пользователь не мог посмотреть документы со смартфона и был вынужден просить агента отправить нужную заметку.

- **Проблема с контентом в больших объёмах**: У автора накопилось более тысячи единиц контента (постов, видео) в Evernote. Использование всего этого контента в качестве контекста для агента было неудобно и требовало перебора большого объёма информации для поиска нужного материала.

- **Граф связей в Obsidian**: Obsidian содержит функцию «граф», которая показывает взаимосвязи между заметками. Это позволяет агенту искать не все заметки подряд, а только те, которые связаны с определённой темой (например, N8n или Cloud Code).

- **Использование Obsidian-агента (библиотекаря)**: Внутри Cloud Code создан отдельный E-агент, называемый «библиотекарем», который работает с Obsidian. Вместо загрузки всего контекста в основной агент, библиотекарь извлекает только нужные заметки и фильтрует лишнее.

- **Детальная статистика видео в Obsidian**: По каждому видео планируется хранить подробную информацию: описание, дату публикации, количество просмотров, CTR, изображение, ссылки, ключевые слова. Это позволит оптимизировать следующие видео на основе опыта предыдущих.

- **Обучение на примерах прошлого контента**: Если определённый заголовок, описание или тип контента не сработал раньше, агент не будет использовать такой подход в будущем. Он анализирует, какие направления контента лучше заходят аудитории.

- **Кеширование как промежуточное решение**: Кеширование снижает стоимость обработки токенов в 10 раз, но это лишь промежуточное решение. Obsidian обеспечивает экономию до 95% токенов благодаря структурированию информации.

- **Разница между папкой файлов и Obsidian**: Хотя Obsidian и простая папка содержат одинаковый объём данных, Obsidian предоставляет структуру и показывает связи между файлами. Это позволяет агенту читать только нужные 10-50 постов вместо сканирования тысячи файлов.

- **Экономия токенов благодаря структурированию**: Тысяча постов — это примерно 300-500 тысяч токенов. При использовании Obsidian с фильтром по теме (например, N8n) агент может обойтись только 5% от этого объёма, достигая экономии в 95%.

- **Синхронизация Obsidian между устройствами**: Если настроена синхронизация, то Obsidian доступен не только на компьютере, но и на мобильном приложении. Пользователь может добавлять и редактировать заметки со смартфона.

- **Практический пример: оптимизация видео номер 1**: Автор демонстрирует файл с оптимизацией одного видео, содержащий 8 шагов, сделанные ChatGPT, Gemini и Claude. Он просит Cloud Code навести порядок в заметке, объединив три варианта в один удобный формат.

- **Структурирование оптимизации в Obsidian**: Cloud Code расставляет теги, даты, названия, объединяет лучшие варианты описаний, SEO-слова, предлагает 10 вариантов заголовков, хештеги, названия для превью, краткое описание и план нарезки видео.

- **Система обучения агента на примерах**: Чем более структурированной и полной информацией располагает агент в Obsidian, тем лучше он может обучаться на своих прошлых результатах и адаптировать свою работу к предпочтениям пользователя.

## Инструменты и технологии

**Obsidian** — бесплатное приложение для хранения заметок в Markdown-формате с функцией графа связей между файлами. Используется как интеллектуальная база знаний для AI-агентов с поддержкой синхронизации между устройствами. [https://obsidian.md]

**Claude Code (Claude Coda)** — расширенная версия Claude с возможностью выполнения кода. Интегрируется через Telegram для удобства, используется как основной E-агент для создания контента и выполнения сложных задач.

**N8n** — платформа для автоматизации и создания workflow'ов. Подключается через MCP к Claude Code для делегирования рутинных задач (просмотр видео, создание субтитров, оптимизация видео). Позволяет снизить расходы на токены за счёт выполнения работы отдельными sub-агентами.

**MCP (Model Context Protocol)** — протокол для подключения инструментов к Claude. Используется для связи Claude Code с N8n и Obsidian, позволяя создавать отдельных агентов для специфических задач.

**Gemini и ChatGPT (через API)** — используются как альтернативные LLM-модели для создания вариантов оптимизации контента (субтитры, описания, заголовки). Подключаются через N8n для параллельной обработки.

**Telegram** — используется как интерфейс для Claude Code, позволяя пользователю писать задачи прямо в мессенджере без необходимости работать в терминале.

**Evernote** — инструмент, где автор хранит контент перед внедрением Obsidian. Содержит более тысячи единиц контента (постов, идей) для различных платформ.

**SuperBase (базы данных)** — использовалась раньше для хранения контента в векторном формате, позволяя агентам обращаться к информации. Заменена на Obsidian как более эффективное решение.

**VDS (Virtual Dedicated Server)** — сервер, на котором хранились локальные папки с контентом. Информация была недоступна с других устройств до внедрения Obsidian.

## К действию

**Оценить текущие расходы на AI-подписку**: Проанализировать, используется ли Claude Max без необходимости. Для большинства задач достаточно подписки Plus (17-20 долларов) в сочетании с оптимизацией архитектуры агентов.

**Внедрить интеграцию Claude Code через Telegram**: Настроить подключение Claude Code к Telegram, чтобы делегировать задачи прямо из мессенджера без использования терминала. Это снижает барьер входа для непрограммистов.

**Создать иерархию агентов с использованием N8n и MCP**: Разделить основного агента на несколько sub-агентов через N8n. Рутинные и однотипные задачи (просмотр видео, создание субтитров, оптимизация контента) должны выполняться отдельными агентами.

**Внедрить Obsidian как систему хранения памяти**: Установить Obsidian, настроить синхронизацию между устройствами, начать собирать весь контент в единую структурированную базу. Это позволит агентам работать с информацией эффективнее.

**Создать библиотекаря (Obsidian-агента)**: Внутри основного Claude Code создать отдельного E-агента, который будет запрашивать информацию из Obsidian, фильтровать результаты и возвращать только релевантные данные основному агенту.

**Структурировать контент по тегам и связям**: При добавлении заметок в Obsidian использовать теги, даты, названия, внутренние ссылки. Это позволит использовать функцию графа для быстрого поиска связанного контента.

**Собрать базу данных по результатам контента**: Хранить в Obsidian детальную статистику по каждому видео (просмотры, CTR, заголовки, описания, ключевые слова). Использовать эту информацию для оптимизации будущего контента.

**Запустить параллельную обработку через несколько LLM**: Использовать Claude, Gemini и ChatGPT через API одновременно для создания разных вариантов оптимизации контента. Обходится в 1-2 рубля за задачу и даёт лучший результат.

**Документировать процессы оптимизации**: Создавать подробные инструкции для каждой внедрённой функции и сохранять их в Obsidian. Это позволит быстро масштабировать систему и обучать новых агентов.

**Просить агента наводить порядок в заметках**: При добавлении контента из разных источников просить основного агента структурировать информацию, расставлять теги, объединять лучшие варианты. Это повысит качество и организованность базы.

**Мониторить экономию токенов**: Отслеживать, как внедрение Obsidian и sub-агентов влияет на расходы. Цель — снизить затраты на 80-95% для рутинных задач при сохранении качества результатов.
