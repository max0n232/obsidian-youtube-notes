## Краткое содержание
Разбор supply chain атаки на Axios (174K зависимых проектов, 100M+ загрузок/неделю). Атакующий получил NPM access token мейнтейнера, добавил одну зависимость (`plain-crypto.js`) в package.json, которая через post-install скрипт устанавливает RAT (Remote Access Trojan) за 1.1 секунды и самоуничтожается.

**Анатомия атаки:**
1. Компрометация long-lived NPM access token мейнтейнера (Jason)
2. Смена email аккаунта на `istop@proton.me`
3. Добавление невинной зависимости `plain-crypto.js` в package.json
4. Pre-stage чистой версии за 18 часов до атаки (bypass review)
5. Публикация через NPM CLI (минуя CI/CD guardrails)
6. Заражённые версии: **1.14.1** и **0.3.04** (в пределах 39 минут)
7. Post-install → dropper (`setup.js`) → XOR + Base64 deobfuscation → detect OS → download RAT from C2
8. Cleanup: удаление setup.js, подмена package.json чистой версией

**Проверка заражения:**
```bash
npm list -g axios  # проверить версии
# Поиск RAT на системе — команды зависят от OS
# C2 IP уже down, но проверить сетевые логи
```

## Применимость к нашей архитектуре

### 1. КРИТИЧНО: n8n + npm dependencies
- n8n установлен через Docker, но **custom nodes** тянут npm packages
- ClaudeClaw: `grammy`, `better-sqlite3`, `@anthropic-ai/sdk` — каждый имеет dependency tree
- **Действие**: audit dependency trees, lock versions, disable post-install scripts

### 2. Защита VPS (Hetzner)
- На VPS работает ClaudeClaw, video-service, n8n — все с npm
- `npm config set ignore-scripts true` — глобально отключает post-install (ломает некоторые пакеты, но защищает)
- Альтернатива: `--ignore-scripts` при install + явный `npm run prepare` для доверенных

### 3. Паттерн атаки применим к MCP серверам
- Мы подключаем MCP серверы через `npx` (firecrawl-mcp, obsidian-mcp) — каждый запуск = fresh install
- Если upstream MCP пакет скомпрометирован → мгновенный доступ к системе с правами Claude Code
- **Действие**: pin versions в settings.json, не использовать `@latest`

### 4. Правила для наших скриптов
- `scripts/package.json` — проверить lock file, запинить версии
- Не использовать `^` ranges для критических зависимостей
- Периодически: `npm audit` + `socket.dev` для supply chain мониторинга
