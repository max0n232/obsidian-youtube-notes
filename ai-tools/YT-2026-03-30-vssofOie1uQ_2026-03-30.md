## Краткое содержание

Видео посвящено подключению экосистемы Google Workspace к Claude Code (AI-агенту на базе Claude от Anthropic) через официальный инструмент от Google. Автор — Григорий Зуевич — показывает пошаговый процесс настройки Google Cloud Console, выдачи гранулярных прав доступа к отдельным сервисам Google (Drive, Sheets, Docs, Slides, Calendar, YouTube Analytics) и последующей авторизации агента. Особый акцент делается на безопасности: автор намеренно не даёт агенту доступ к Gmail и ограничивает права на YouTube только аналитикой без возможности редактирования. Отдельно разбирается нюанс работы Claude Code через Telegram-бота (а не через стандартный терминал), а также проблема слёта авторизации каждые 7 дней в тестовом режиме OAuth и способ её решения. Видео рассчитано на людей, уже знакомых с Claude Code и желающих расширить его возможности за счёт интеграции с Google-сервисами.

## Ключевые моменты

- **Google выпустил официальный инструмент для подключения Workspace к Claude Code.** Интеграция позволяет AI-агенту работать с Gmail, Google Drive, Sheets, Docs, Slides, Calendar, YouTube Analytics и другими сервисами экосистемы Google. Подключение возможно как полное, так и частичное — с точечным выбором сервисов и уровней доступа.

- **Автор использует Claude Code через Telegram-бота, а не через стандартный терминал.** Такой подход он называет «тлод-клоу» или «клод-клоу» — это тот же Claude Code, но весь интерфейс взаимодействия вынесен в Telegram. По мнению автора, это значительно удобнее, чем работа напрямую через терминал.

- **Подключение через Telegram-бота может вызывать проблемы с авторизацией OAuth.** Автор столкнулся с тем, что агент завис на 20 минут при попытке авторизоваться через Telegram-интерфейс. Решение — часть шагов выполнять напрямую через терминал, особенно на этапе первичной авторизации.

- **Первый шаг настройки — создание нового проекта в Google Cloud Console.** Необходимо перейти в Google Cloud Console, создать новый проект (автор называет его GWS Cloud Code), после чего перейти в раздел API & Services для дальнейшей настройки.

- **Включение нужных API через раздел Library в API & Services.** В библиотеке API нужно найти и включить (Enable) каждый сервис, к которому планируется дать доступ: Google Drive API, Google Sheets API, Google Docs API, Google Calendar API, YouTube Analytics API, Google Slides API и другие. Gmail API автор намеренно не включает, чтобы не давать агенту доступ к электронной почте.

- **Настройка OAuth Consent Screen — важный шаг для авторизации.** В разделе API & Services необходимо перейти в OAuth Consent Screen, задать название приложения (например, GWS Cloud Code), указать рабочий email. Для неверифицированных организаций и обычных пользователей выбирается тип External.

- **Гранулярная настройка прав доступа через Data Access.** В разделе Data Access для каждого подключённого сервиса можно выбрать конкретный уровень доступа: только чтение, чтение и редактирование, удаление и т.д. Автор, например, даёт полный доступ к Google Sheets, но для YouTube Analytics ограничивается только правом просмотра и выгрузки аналитики без возможности вносить изменения на канале.

- **Создание OAuth Client ID в формате Desktop App и скачивание JSON-файла.** После настройки прав необходимо создать учётные данные (Create Client), выбрав тип Desktop App (не Web Application, как при работе с N8n). Скачанный JSON-файл содержит данные для авторизации и передаётся агенту.

- **JSON-файл помещается в специальную папку на сервере — `config/gws`.** Агенту можно дать команду создать папку самостоятельно и переместить туда JSON-файл, не делая это вручную. Автор демонстрирует, как это делается через Telegram-чат с агентом одной командой.

- **Авторизация через ссылку OAuth требует ручного подтверждения в браузере.** Агент генерирует ссылку для авторизации, по которой нужно перейти в браузер, выбрать нужный Google-аккаунт, нажать Advanced → Continue (обходя предупреждение о непроверенном приложении), выбрать разрешения и получить финальную ссылку-код. Эту ссылку нужно передать обратно агенту для завершения авторизации.

- **Предупреждение «Приложение не прошло проверку» обходится публикацией в режиме тестирования.** Google показывает предупреждение для неверифицированных приложений. Нужно перейти в раздел Audience, опубликовать приложение (оставив его в тестовом режиме), после чего авторизация проходит через кнопку Advanced → Continue.

- **Ключевая проблема тестового режима — авторизация слетает каждые 7 дней.** Это та же проблема, что возникает при работе с N8n. Чтобы авторизация была постоянной, необходимо перевести проект из тестового режима в рабочий (Production) в настройках OAuth Consent Screen.

- **После авторизации агенту нужно явно сообщить о доступных инструментах.** По умолчанию агент не знает, что у него появился доступ к Google-сервисам. Необходимо дать ему соответствующую инструкцию или системный промпт, после чего он начинает использовать новый инструментарий.

- **Проверка работоспособности — создание файлов и папок в Google Drive.** Автор просит агента создать Google-таблицу и папку с именем «Cloud Code» в Drive. Агент успешно выполняет задачу, создаёт папку, таблицу внутри неё и возвращает прямую ссылку для доступа к документам.

- **Агент получает не только доступ, но и встроенные навыки работы с каждым сервисом.** Claude Code умеет работать с Google Формами, отвечать на электронные письма, сохранять данные со встреч в Calendar, управлять таблицами и документами. То есть интеграция даёт полноценный функциональный набор, а не просто технический доступ.

- **Настройка Key Forwarding в Termius позволяет автоматизировать авторизацию.** Для тех, кто хочет избежать ручного копирования ссылки авторизации, можно настроить Key Forwarding в SSH-клиенте Termius, прописав специальные значения портов (37, 37, 3, 227 и др.). Это позволяет агенту авторизоваться самостоятельно без участия пользователя.

- **Рекомендуется использовать модуль IAM (ARM) от Google для безопасности.** Автор упоминает модуль управления доступом IAM как систему безопасности Google и рекомендует его устанавливать наряду с основными API для обеспечения безопасной работы агента с инструментами.

## Инструменты и технологии

- **Claude Code (Anthropic)** — AI-агент на базе Claude, который используется как основной исполнитель задач. В данном видео подключается к экосистеме Google Workspace. Автор работает с ним через Telegram-интерфейс.

- **Google Cloud Console** — [console.cloud.google.com](https://console.cloud.google.com) — платформа для управления Google Cloud проектами. Используется для создания проекта, включения API и настройки OAuth-авторизации.

- **Google Workspace APIs** — набор API от Google: Drive API, Sheets API, Docs API, Slides API, Calendar API, YouTube Analytics API, Gmail API (опционально), Google Analytics, Google Search Console, AdSense API. Каждый API включается отдельно в Library раздела API & Services.

- **OAuth 2.0 / OAuth Consent Screen** — стандарт авторизации Google, используемый для выдачи прав доступа приложению. Настраивается в Google Cloud Console, тип — External, Desktop App Client ID.

- **Telegram Bot** — используется как интерфейс для взаимодействия с Claude Code вместо стандартного терминала. Автор описывал его настройку в отдельном предыдущем видео.

- **Termius** — SSH-клиент для подключения к серверу. Упоминается в контексте настройки Key Forwarding для автоматизации OAuth-авторизации.

- **GitHub** — репозиторий с официальной инструкцией от Google по подключению GWS к Claude Code. Ссылку на репозиторий агенту можно передать напрямую, и он сам выполнит все шаги по инструкции.

- **IAM (Identity and Access Management)** — система управления доступом Google Cloud. Рекомендуется к установке как дополнительный уровень безопасности.

- **N8n** — упоминается как пример другого инструмента, при работе с которым возникала аналогичная проблема слёта OAuth-авторизации каждые 7 дней в тестовом режиме.

## Цитаты

> Ребята из Google выкатили инструмент, который позволяет буквально в один клик, либо в один промпт подключить экосистему Google к лод-коду.

> Мне нравится работать с кодом именно через Telegram, а не через терминал. На мой взгляд, это намного удобнее, но это всё тот же клод-код.

> Я ему не дал доступ к Gmail. Я ему дал частичный доступ к своему YouTube-каналу, например, только для анализа моих видео. То есть я его частично ограничил, дал ему функционал только смотреть, редактировать, корректировать, но, например, не удалять никакие документы.

> Здесь я вам покажу не просто, как подключить к инструментарию Google, а как в первую очередь безопасно подключить.

> Если вы все это будете делать через Telegram-бота — у меня он завис на 20 минут. Я думал, что он выполняет работу, потом зашёл в терминал и спросил, как у него дела, он сказал, что не может авторизоваться.

> Если мы его не выключим из режима тестирования, то у нас авторизация будет слетать каждые 7 дней. С этим мы с вами когда-то сталкивались при работе с сервисом N8n. Здесь эта проблема точно такая же.

> У него также с этим инструментарием есть подробные скиллы и работе с тем или иным инструментом. То есть он отлично может делать форму, он отлично может отвечать на электронные письма, он может сохранять данные со встреч, работать с календарём. Полный функционал для работы с экосистемой Google у него теперь есть.

> Нам необходимо выбрать именно Desktop App. Грубо говоря, как будто он у нас работает на нашем компьютере. Не Web Application — когда мы использовали N8n, у нас был Web Application, потому что мы создавали приложение с обратным запросом.

## К действию

- [ ] **Создать отдельный Google-аккаунт** для работы агента, чтобы не смешивать личные данные с данными AI-агента и иметь чистую среду для экспериментов.
- [ ] **Зайти в Google Cloud Console**, создать новый проект специально под Claude Code интеграцию, назвать его осмысленно (например, GWS-ClaudeCode).
- [ ] **Составить список API**, которые реально нужны агенту, и включить только их. Не включать Gmail API без крайней необходимости — это снижает риски несанкционированного доступа к переписке.
- [ ] **Настроить OAuth Consent Screen** с типом External, добавить необходимые email-адреса тестовых пользователей.
- [ ] **В разделе Data Access настроить гранулярные права** для каждого сервиса: для YouTube Analytics — только чтение, для Drive и Sheets — по необходимости, но без права удаления на начальном этапе.
- [ ] **Создать OAuth Client ID типа Desktop App**, скачать JSON-файл и передать его агенту с командой разместить в папку `config/gws`.
- [ ] **Выполнить авторизацию через браузер**: перейти по ссылке от агента, пройти через Advanced → Continue, передать финальный URL обратно агенту.
- [ ] **Перевести проект из тестового режима в Production** в настройках OAuth Consent Screen (раздел Audience → Publish App), чтобы авторизация не слетала каждые 7 дней.
- [ ] **Передать агенту явную инструкцию** о том, что у него теперь есть доступ к Google Workspace — без этого он не будет использовать новый инструментарий.
- [ ] **Проверить работу интеграции** командой создания тестовой папки и Google-таблицы в Drive, убедиться что агент возвращает рабочие ссылки на созданные документы.
- [ ] **Скачать инструкцию** из Telegram-канала автора (первое закреплённое сообщение) — там задокументированы все шаги включая настройку Key Forwarding в Termius для автоматической авторизации.
- [ ] **Посмотреть предыдущее видео автора** о подключении Claude Code к Telegram, если этот шаг ещё не сделан — без него Telegram-интерфейс работать не будет.
