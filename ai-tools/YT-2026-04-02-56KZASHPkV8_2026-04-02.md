## Краткое содержание

Видео от Игоря Зуевича посвящено решению одной из главных проблем при использовании AI для создания контента — несоответствию генерируемых текстов личному стилю, голосу и философии автора. Главный тезис состоит в том, что вместо установки «скиллов лучших копирайтеров» нужно обучать AI-агента на собственном реальном контенте, выгруженном из социальных сетей. Автор демонстрирует конкретную техническую систему: экспорт постов из Telegram в JSON, их структурирование в Obsidian через Markdown-заметки с тегами и перелинковкой, а затем использование этой базы знаний AI-агентом для выявления персональных паттернов написания. Система подходит как для Telegram, так и для Instagram, Threads, TikTok и Reels. Видео рассчитано на авторов и предпринимателей, которые уже используют AI-агентов (Claude Code, Codex, ChatGPT, Telegram-боты) для создания контента, но недовольны его качеством и обезличенностью.

## Ключевые моменты

- **Главная проблема AI-контента — отсутствие личного голоса.** Большинство пользователей разочарованы контентом, который генерирует AI: он не подходит под их темперамент, стиль и философию ведения социальных сетей. Это актуально для любых моделей — Claude Sonnet, Codex, ChatGPT 5, Telegram-ботов и агентов на сервере.

- **Ошибка №1 — надежда на «скиллы лучших копирайтеров».** Многие думают, что если установить агенту навыки топовых копирайтеров, он сразу начнёт писать качественный контент. На практике такой контент не набирает просмотров и лайков, выглядит чужеродным и ощущается как «написанный бред».

- **Правильный подход — обучение на собственном контенте.** Автор предлагает принципиально иной метод: выгрузить весь собственный реальный контент из социальных сетей и предоставить его AI-агенту для анализа паттернов. Только так агент начинает понимать, как именно думает и пишет конкретный автор.

- **Документация системы в трёх файлах.** Вся система задокументирована в трёх файлах: «Полная стратегия создания контента на март 2026 года», «Полная система генерации контента с помощью AI-моделей» и связанные материалы. Файлы доступны в Telegram-боте автора в форматах MD, PDF и DOCX.

- **Экспорт контента через десктопную версию Telegram.** Для выгрузки постов из Telegram-канала рекомендуется использовать не мобильное приложение, а отдельную десктопную версию (Telegram Desktop), которая скачивается с официального сайта. В ней доступна функция «Экспорт истории чата», позволяющая выгрузить все сообщения от первого до текущего дня в формате JSON.

- **Формат JSON как основа для дальнейшей обработки.** Выгруженный JSON-файл содержит все посты канала, включая текстовые и видео-публикации. Этот файл передаётся AI-агенту (Claude Code, Codex или боту, подключённому к Obsidian), который разбирает контент и структурирует его в базе знаний.

- **Obsidian как центральная база знаний для AI-агента.** Система строится на инструменте Obsidian, позволяющем хранить заметки в формате Markdown — удобном для AI-агентов. Для каждой социальной сети создаётся отдельная папка (Instagram, Telegram, Threads, TikTok), и весь контент размещается там в структурированном виде.

- **Структура каждой заметки-поста в Obsidian.** Каждый пост получает: уникальный идентификатор, дату публикации, название, теги, направление/тематику, источник (например, канал Игоря Зуевича), данные о реакциях из JSON-файла, полный текст поста и перелинковку с похожими постами на ту же тематику.

- **Граф Obsidian как навигационная система для агента.** AI-агент использует граф связей между заметками для эффективной навигации в базе контента без необходимости загружать весь объём в контекст. Это существенно экономит токены при генерации нового контента, так как агент находит релевантные материалы по ключевым словам и паттернам.

- **Кросс-платформенное создание контента.** Система позволяет создавать контент для одной платформы на основе базы другой. Например, для Threads агент создаёт контент на основе Telegram-базы, не имея изначально отдельной базы Threads. При выборе пользователем одного из трёх предложенных вариантов агент добавляет его в базу Threads и начинает выявлять паттерны, релевантные именно этой платформе.

- **Итеративное улучшение через обратную связь.** Первые посты, которые создаёт агент, могут не полностью удовлетворять автора — их нужно корректировать вручную. Автор объясняет агенту, что именно не нравится (например, «слишком много английских слов», «нужен более человеческий язык»), и агент фиксирует эти предпочтения как правила. После пяти-десяти постов с обратной связью агент начинает создавать контент, максимально близкий к стилю автора.

- **Автоматическое обновление базы новыми постами.** После создания нового поста его не нужно заново выгружать через десктопный Telegram — достаточно отправить агенту текст или скриншот, и он самостоятельно добавит пост в Obsidian-базу. Через несколько дней можно также передать данные о реакциях на пост, чтобы агент учитывал их при анализе паттернов эффективности.

- **Перспектива — автономный агент в Telegram-канале.** Автор упоминает, что следующий шаг — «поселить» агента непосредственно в Telegram-канал, чтобы он автоматически отслеживал реакции на посты и самостоятельно обновлял базу знаний без ручного участия. На момент записи видео этот этап ещё не реализован.

- **Условие работы системы — наличие собственного голоса бренда.** Система эффективна только при наличии реального авторского контента, написанного самостоятельно. Если своего контента нет, можно спарсить канал понравившегося автора и обучить агента на нём, однако автор подчёркивает, что в таком случае контент будет не вашим, и рекомендует хотя бы частично писать самому и корректировать результаты агента.

- **Применимость системы ко всем форматам и платформам.** Описанная архитектура применима не только к текстовым постам, но и к видеоконтенту (Reels, TikTok, YouTube Shorts). Структура папок в Obsidian позволяет организовать работу сразу по всем социальным сетям в единой системе.

## Инструменты и технологии

- **Obsidian** — инструмент для создания и хранения заметок в формате Markdown. Используется как центральная база знаний AI-агента: хранит весь структурированный контент, обеспечивает граф связей между постами и позволяет агенту эффективно искать релевантные материалы без перегрузки контекста.

- **Telegram Desktop (десктопная версия Telegram)** — отдельное приложение, скачиваемое с официального сайта Telegram. Используется для экспорта истории чата/канала в формате JSON, что недоступно в стандартном мобильном или десктопном приложении Telegram.

- **Claude Code / Claude Sonnet** — AI-модели от Anthropic. Упоминаются как варианты AI-агентов, которых можно использовать для разбора и структурирования контента, а также для генерации новых постов на основе выявленных паттернов.

- **Codex (OpenAI)** — AI-модель от OpenAI. Упоминается как альтернативный вариант AI-агента для той же системы создания и анализа контента.

- **ChatGPT** — упоминается как одна из платформ, на которых работают AI-агенты для создания контента. Используется в том числе через стандартный чат для генерации постов.

- **Telegram-боты / Claude Bot / Codex Bot** — AI-агенты, работающие на базе моделей Claude или Codex и интегрированные в Telegram. Позволяют взаимодействовать с AI-агентом через мессенджер, получать готовые посты и давать обратную связь в режиме диалога.

- **Формат JSON** — формат выгрузки истории Telegram-канала. Содержит все посты с метаданными (даты, реакции, типы сообщений), которые AI-агент разбирает и конвертирует в Markdown-заметки для Obsidian.

- **Формат Markdown (MD)** — формат хранения заметок в Obsidian. Удобен для AI-агентов, так как позволяет легко добавлять структурированные метаданные (теги, даты, перелинковки) и парсить содержимое.

## Цитаты

> Когда мы создаем контент с помощью искусственного интеллекта, мы часто разочаровываемся в том контенте, который он для нас создаёт. То есть, он либо не пригоден, либо он совсем не подходит под наш темперамент, под наш стиль, под вообще нашу философию ведения контента в социальных сетях.

> Все надеются вот на эту штуку, которая называется скиллы, и думают, что если сейчас я ему установлю скиллы лучших копирайтеров, то он тут же станет писать именно как лучшие копирайтеры, а потом этот контент нигде не набирает просмотров, нигде не набирает лайков, и вообще он мусорный.

> Я рекомендую вам подойти к обучению создания контента для своего AI-бота немного иначе.

> Он работает с парсингом сохранения, то есть каждый пост становится отдельным Markdown форматом. Когда я написал какой-то пост, мне уже не нужно выгружать его сюда, я его просто закидываю либо скриншотом, либо просто текстом, и он его добавляет уже в нашу базу знаний.

> После пятого, седьмого, десятого поста у вас уже будет, Claude Code, либо Claude Bot, которого мы с вами создаём через Telegram, он будет уже точно понимать, что вы хотите, как вы мыслите, как вы пишете, какой тип контента удовлетворяет именно вас.

> Вот такая вот простая схема позволяет вам решить проблему с контентом навсегда, но, конечно же, при условии, что у вас уже есть голос бренда, то есть у вас уже есть чётко сформулированный контент, который вы пишете сами.

> Если такого нет, то вы можете спарсить любой Telegram-канал, вытащить оттуда весь контент того автора, который вам нравится, и создавать контент на его основе, но, конечно же, это будете не вы. Поэтому лучше, конечно, писать частично самому, исправлять, усиливать.

> Самое интересное, когда он работает с Obsidian, он на создание этого контента не тратит большое количество токенов, потому что за счёт этого графа он может ориентироваться в контенте и выбирать то, что нужно.

## 🤖 ARKHOS / AI-Агенты

### PATTERN — Obsidian как долгосрочная память агента с навигацией через граф
Вместо загрузки всего контекста в prompt, агент работает с Obsidian-графом: находит релевантные заметки по ключевым словам и перелинковке, подгружает только нужные фрагменты. Экономия токенов существенная — граф заменяет векторный поиск для небольших баз (~сотни заметок). Применимо как Ghost Session Memory в ARKHOS: граф связей между сессиями вместо монолитного контекста.

### FACT — Схема Markdown-заметки для поста как структура памяти агента
```markdown
---
id: post_001
date: 2024-03-15
title: "Название поста"
tags: [тема, формат, платформа]
direction: маркетинг
source: telegram/channel_name
reactions: {likes: 42, views: 1200}
---

Полный текст поста...

## Связанные посты
- [[post_003]] — та же тематика
- [[post_007]] — схожий формат
```
Эта структура читается агентом без парсинга, теги работают как фильтры при поиске паттернов.

## ✍️ AiGeneration / SocialMedia

### GOTCHA — «Скиллы лучших копирайтеров» не работают без личной базы
Промпты в стиле «пиши как Хемингуэй / топ-копирайтер» дают обезличенный контент, который не набирает реакций. Причина: модель оптимизирует под усреднённый стиль, а не под конкретную аудиторию автора. Единственный рабочий вариант — дать модели реальные примеры именно вашего контента с данными об эффективности.

### PATTERN — Итеративная калибровка стиля через явные правила (5–10 итераций)
1. Генерируем 3 варианта поста
2. Выбираем ближайший к нужному
3. Объясняем **конкретно**, что не так в остальных (не «плохо», а «слишком много английских слов», «нет личной истории», «слишком формальный тон»)
4. Агент фиксирует это как правило в своей памяти / системном промпте
5. После 5–10 итераций агент выходит на целевой стиль

Применимо для настройки любого LLM-агента под конкретного пользователя или бренд.

### PATTERN — Кросс-платформенный bootstrap через существующую базу
Для новой платформы (например, Threads) отдельной базы нет. Алгоритм:
1. Агент генерирует контент на основе Telegram-базы
2. Пользователь выбирает 1 из 3 вариантов
3. Выбранный пост добавляется в папку `/Threads` как первый образец
4. После 10–15 постов агент начинает различать платформенную специфику

Так строится база с нуля без ручного написания.

## 📦 Техническая реализация (ARKHOS + AiGeneration)

### FACT — Telegram Desktop экспортирует историю канала в JSON, мобильное приложение — нет
Только `Telegram Desktop` (desktop.telegram.org, не веб-версия) имеет функцию **«Экспорт истории чата»**. Формат: JSON с полями `text`, `date`, `reactions`, `type`. Этот файл передаётся агенту для первичного парсинга и структурирования в Obsidian.

### PATTERN — Инкрементальное обновление базы без повторного экспорта
После первичного импорта новые посты добавляются напрямую:
- Текстом в чат агента → агент создаёт заметку сам
- Скриншотом → агент парсит через vision

Через 2–3 дня отправляем данные о реакциях → агент дополняет поле `reactions` в заметке и пересчитывает паттерны эффективности. Это закрывает проблему актуальности базы без DevOps-инфраструктуры.

### EXAMPLE — Промпт для первичного парсинга JSON-экспорта в Obsidian
```
У меня есть JSON-файл с экспортом Telegram-канала. 
Для каждого сообщения типа "message" создай отдельный 
Markdown-файл в папке /Telegram/ со структурой:
[вставить структуру из FACT выше]

Перелинкуй посты с одинаковыми тегами через [[wiki-links]].
Создай индексный файл /Telegram/INDEX.md с графом тематик.
```
Работает с Claude Code (с доступом к файловой системе через MCP или хуки).

> **Ключевой принцип всей системы:** LLM не моделирует «хорошего автора» абстрактно — она моделирует конкретного автора по корпусу его текстов + фидбек по эффективности. Чем больше размечен корпус (реакции, темы, форматы), тем точнее модель.
