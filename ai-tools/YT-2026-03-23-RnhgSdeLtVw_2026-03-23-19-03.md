## Краткое содержание

Видео рассказывает о создании безопасного моста между мессенджером Telegram и экосистемой Claude Code, позволяющего использовать официальный Claude как персональный AI-ассистент через удобный интерфейс телефона. Автор демонстрирует, почему этот подход значительно безопаснее и надёжнее, чем использование OpenAI Bot, и показывает пошаговый процесс развёртывания бота за счёт одного мегапромта из 900+ строк кода. Видео предназначено для пользователей Claude Code, разработчиков, автоматизаторов и тех, кто хочет создать собственного AI-ассистента без рисков блокировки и утечки данных.

## Ключевые моменты

- **Основная идея проекта**: Создание официального моста между Claude Code (инструмент для программистов, работающий через CLI в терминале) и мессенджерами (Telegram, WhatsApp, iMessage), превращающего Claude в персонального ассистента, доступного с телефона. Это не отдельный бот, а интеграция существующей мощной системы с удобным интерфейсом.

- **Проблемы OpenAI Bot**: OpenAI Bot активно блокируют различные сервисы (Gemini, Google, ChatGPT) за превышение лимитов и нарушение правил безопасности. Даже при использовании API-ключей и платных подписок происходят блокировки аккаунтов. Бот хранит все данные в своей системе, что создаёт риск утечки конфиденциальной информации.

- **Преимущества Claude Code подхода**: Экосистема Claude намного быстрее развивается, имеет больше возможностей, чем OpenAI Bot. Официальная инфраструктура Claude защищена и безопасна, так как бот остаётся внутри экосистемы и не передаёт данные третьим сторонам. Невозможно забанить официального Claude при правильном подключении.

- **Архитектура моста**: Мост работает как адаптер каналов, превращая Telegram в кнопку входа к Claude Code на вашем компьютере или сервере. Два разных мира (CLI-инструменты Claude и внешний мессенджер) соединяются слоем интеграции, позволяющим взаимодействовать с Claude через удобные инструменты без дублирования функций.

- **Контроль приватности и безопасность**: Весь мозг системы (файлы, сессии, правила) остаётся там, где он расположен, а Telegram служит просто интерфейсом. Данные не передаются и не могут быть украдены третьими лицами. API-токены остаются внутри Claude Code и не извлекаются наружу, что отличается от подхода OpenAI Bot.

- **Инструмент развёртывания — Мегапромт**: Один промт из 900+ строк кода содержит полную инструкцию для Claude по развёртыванию всей инфраструктуры бота. Промт существует в английском языке (для быстрого развёртывания) и переводе на русский (для понимания процесса). После загрузки этого файла в Claude или ChatGPT, система сама разворачивает бота и подключает его к Telegram.

- **Трёхуровневая система памяти**: Первый уровень — возобновляемые сессии с идентификатором Session ID, продолжающие контекст в одном диалоге. Второй уровень — разделённая долгосрочная и затухающая память. Третий уровень — поиск по памяти перед каждым ответом с добавлением релевантного контекста. Вся память хранится локально на сервере без использования внешних сервисов.

- **Процесс первичной настройки**: После запуска мегапромта Claude задаёт несколько вопросов: выбор платформы (Discord, Telegram, iMessage, WhatsApp), нужен ли голос, уровень использования памяти и опциональные функции (запуск задач по расписанию, автоматический запуск при старте сервера через Systemd). Весь процесс занимает примерно 13-14 минут.

- **Авторизация и идентификация**: После развёртывания бот требует подтверждения через Telegram, где нужно отправить команду получения чат-ID. Бот запоминает ваш уникальный идентификатор и дальше взаимодействует только с вами как с создателем и главным пользователем.

- **Универсальность подхода**: Один и тот же мегапромт может быть адаптирован для работы не только с Claude Code, но и с ChatGPT Codex (экосистема ChatGPT) и Gemini (экосистема Google). Все агенты будут обладать инструментами и возможностями своих соответствующих сервисов.

- **Инструменты и интеграции**: Бот поддерживает подключение различных сервисов: Eleven Labs для голоса, Grok для расширенного функционала, и все встроенные инструменты Claude Code. Можно добавлять новые скиллы, навыки, интеграции через MCP-серверы и локальные файловые системы.

- **Файловая структура и развёртывание**: Используется Visual Studio Code с расширением Remote SSH для удобного доступа к файлам на сервере. Мегапромт размещается в отдельной папке внутри Claude Code директории (например, `/cloudbot/`). После загрузки файла в Claude через терминал, система автоматически создаёт всю необходимую инфраструктуру.

- **Документирование CloudMD**: Система создаёт файл инструкций CloudMD, где описываются характеристики бота, его возможности, правила и знания. Этот файл можно заполнить самостоятельно или с помощью Claude, и впоследствии бот будет обладать всеми описанными в нём знаниями и навыками.

- **Процесс диагностики ошибок**: При возникновении проблем (например, бот не видит сообщения в Telegram) Claude может самостоятельно анализировать логи и исправлять ошибки. В примере автора проблема была в несогласованности между основной директорией Claude и новой директорией бота, которую Claude автоматически разрешил.

- **Использование подписки Claude**: Работа бота зависит от лимитов подписки Claude Code. Если использовать стандартную (не максимальную) подписку, есть ограничения на количество операций, поэтому нужно следить за расходом токенов и планировать использование. Автор в видео исчерпал свой лимит во время тестирования.

- **Будущее AI-ботов**: Такой подход (официальный мост между мессенджером и профессиональным инструментом) представляет следующий этап развития AI-ассистентов. Вместо создания отдельных ботов с нуля, правильнее использовать уже настроенные системы через удобные интерфейсы, что экономит ресурсы и повышает надёжность.

## Инструменты и технологии

- **Claude Code** — официальная CLI-система от Anthropic для работы с Claude AI, используется как основная база для развёртывания бота. Содержит встроенные инструменты, MCP-серверы для интеграций и систему памяти.

- **Telegram Bot API** — мессенджер, используемый как основной интерфейс для взаимодействия с Claude Code. Бот подключается через официальный API Telegram и работает как промежуточный слой.

- **Visual Studio Code с расширением Remote SSH** — IDE для удобного редактирования файлов на сервере. Расширение Remote SSH от Microsoft позволяет просматривать и редактировать файловую структуру удалённого сервера как локально.

- **Systemd** — инструмент Linux для автоматического запуска бота при старте сервера, обеспечивающий постоянную работу системы без ручного запуска.

- **Мегапромт (900+ строк)** — мета-инструмент в виде развёрнутого текстового промта, содержащий полную инструкцию для Claude по развёртыванию и настройке бота. Доступен в английском и русском вариантах.

- **Eleven Labs** — опциональный сервис для синтеза голоса, может быть подключен для добавления голосовых возможностей боту (автор упомянул проблемы с интеграцией).

- **Grok** — опциональный инструмент для расширения функционала бота (автор обещает тестировать и подключать в будущих видео).

- **ChatGPT Codex (GPT-4 с Code Interpreter)** — альтернативная экосистема, для которой может быть адаптирован тот же мегапромт для создания аналогичного моста.

- **Gemini (Google AI)** — ещё одна альтернативная экосистема для создания моста (автор пока не тестировал).

- **Linux сервер** — инфраструктура для развёртывания системы (используется в примере автора). Может также использоваться macOS или Windows.

## К действию

- **Загрузить мегапромт**: Скачать файл с инструкцией (900+ строк) из первого закреплённого сообщения в комментариях видео. Файл доступен в английской и русской версиях.

- **Выбрать платформу**: Определиться, на какой операционной системе будет развёртываться бот (Linux сервер, macOS или локальный компьютер) и убедиться в наличии SSH-доступа для удалённого сервера.

- **Установить Visual Studio Code с Remote SSH**: Если используется удалённый сервер, установить расширение Remote SSH от Microsoft для удобного управления файлами.

- **Создать отдельную директорию для бота**: В папке Claude Code создать новую папку (например, `cloudbot/`) и разместить там файл мегапромта.

- **Запустить Claude с мегапромтом**: Открыть Claude или ChatGPT, загрузить файл с инструкцией и попросить выполнить все действия по развёртыванию бота согласно инструкции.

- **Ответить на вопросы настройки**: В процессе развёртывания ответить на четыре главных вопроса: выбор платформы (Telegram/WhatsApp/Discord/iMessage), включение голоса, уровень памяти и опциональные функции (автозапуск через Systemd, планировщик задач).

- **Получить чат-ID в Telegram**: После развёртывания получить сообщение в Telegram и отправить команду получения чат-ID для авторизации бота.

- **Проверить подключение**: Написать боту простую команду (например, "ты на связи?") и убедиться в его ответе. Если есть ошибки, передать Claude логи для автоматического исправления.

- **Настроить файл CloudMD**: Заполнить файл инструкций (CloudMD), описав характеристики бота, его возможности и правила работы. Это позволит боту лучше понимать свои задачи и контекст.

- **Подключить дополнительные сервисы**: По мере необходимости подключить дополнительные инструменты (Eleven Labs для голоса, Grok, и другие MCP-серверы) через параметры бота.

- **Мониторить расход токенов**: Регулярно проверять лимиты подписки Claude Code и планировать использование бота в соответствии с доступным количеством токенов.

- **Документировать процесс**: Вести записи об ошибках, решениях и изменениях конфигурации для быстрого воспроизведения или масштабирования системы.

- **Расширить возможности**: Экспериментировать с добавлением новых скиллов, интеграций и функций, используя встроенные возможности Claude Code и MCP-серверов.

- **Рассмотреть альтернативные экосистемы**: При необходимости адаптировать мегапромт для работы с ChatGPT Codex или Gemini для создания аналогичных ботов в других экосистемах.

- **Поделиться результатами**: Опубликовать свой опыт в сообществе, поделиться модификациями мегапромта и улучшениями для помощи другим разработчикам.
