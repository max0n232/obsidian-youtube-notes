## Краткое содержание

Видео представляет собой детальное руководство по правильной настройке е-агентов (искусственных интеллектуальных ассистентов) на основе различных LLM-моделей, включая Claude, GPT и Gemini. Автор объясняет, что независимо от выбранной платформы агента, ключевым фактором успеха является его корректная конфигурация, позволяющая агенту работать с максимальной эффективностью и понимать потребности пользователя с полуслова. Видео предназначено для тех, кто хочет автоматизировать свою работу и создать собственного интеллектуального помощника, работающего через удобные интерфейсы вроде Telegram. Автор демонстрирует пошаговый процесс настройки е-агента Codex на примере собственной системы, используя готовый чек-лист и объясняя каждый этап интеграции с системой управления знаниями Obsidian.

## Ключевые моменты

- **Значимость правильной настройки агента**: Многие пользователи создают е-агентов, но они работают неправильно из-за отсутствия корректной конфигурации. Первый шаг — это не добавление инструментов, а познакомление агента с пользователем и его потребностями.

- **Экономическая целесообразность использования официальных подписок**: Использование официальных подписок от Google, Claude и ChatGPT намного дешевле, чем работа через API. Автор платит всего 17 долларов за подписку на Claude Code вместо нескольких сотен долларов при использовании API.

- **Преимущества интеграции с Telegram**: Размещение е-агентов в Telegram позволяет взаимодействовать с ними через мобильный телефон, отправлять файлы, голосовые сообщения и работать из любого места, а не только из терминала компьютера.

- **Структура пошагового чек-листа**: Процесс настройки разделён на 6 основных блоков: знакомство с агентом, настройка памяти, создание внешней памяти в Obsidian, добавление скиллов и хуков, подключение инструментов (CLI, MCP, WebHook) и финальная оптимизация системы.

- **Первый блок — знакомство с агентом**: Необходимо загрузить историю взаимодействия с различными LLM или использовать специальный промпт для создания досье пользователя. Эта информация структурируется в файл Agent.md, который агент использует как оперативную память.

- **Трёхуровневая система памяти агента**: Первый уровень (Agent.md) — это оперативная память (голова агента), второй уровень (Agent Memory) — долгосрочная память (дневник), третий уровень (Obsidian) — внешняя память (библиотека). Каждый уровень отвечает за разные функции.

- **Использование Obsidian для построения базы знаний**: Obsidian позволяет создавать перелинковки между заметками и использовать графы связей, что помогает агенту быстро находить нужную информацию по ключевым словам без необходимости читать каждую заметку полностью.

- **Оптимизация размера загружаемого контекста**: Чем меньше информации загружается при запуске, тем меньше токенов расходуется. Поэтому используется многоуровневая память: при включении агент загружает основное (Agent.md), при специализированных задачах обращается к другим уровням памяти.

- **Скиллы как инструкции в формате Markdown**: Скиллы представляют собой инструкции, которые агент может использовать для выполнения задач. Они вызываются вручную или автоматически и размещаются в соответствующих папках (CloudSkills, CodexSkills).

- **Хуки как правила и триггеры**: Хуки — это автоматические обработчики событий, которые срабатывают в 100% случаев без требования к LLM. Они устанавливают правила форматирования ответов, добавления эмодзи, выделения важной информации и другие автоматические действия.

- **Источники скиллов и хуков**: Скиллы и хуки можно взять с Marketplace, с официального репозитория Anthropic на GitHub или создать собственные. Автор рекомендует начать со скилла Creators от Anthropic, который позволяет агенту создавать собственные скиллы.

- **Подключение внешних инструментов на пятом блоке**: На этапе подключения инструментов интегрируются Google Workspace (Gmail, Calendar, Google Sheets через CLI), n8n для автоматизации рабочих процессов, FireCrawl для сбора информации, Perplexity, GitHub и другие сервисы.

- **Автоматизация через n8n**: При подключении n8n е-агент может автоматически скрейпить посты, создавать контент для различных платформ, обрабатывать видео и выполнять другие рутинные задачи без участия пользователя.

- **Шестой блок — обновление и доработка**: На заключительном этапе настраиваются автоматические обновления системы, расписание выполнения задач (например, каждый день в определённое время) и дополнительная оптимизация.

- **Взаимодействие нескольких агентов между собой**: Агенты можно устанавливать вместе на одном сервере и позволять им анализировать и улучшать друг друга. Это усиливает их возможности и делает систему более мощной и универсальной.

- **Резервная копия агента**: Автор создаёт резервного е-агента на базе Codex как страховку на случай блокировки основного Claude Code агента, чтобы работа не прерывалась и была непрерывность в выполнении задач.

- **Разница между Claude Code и Codex**: Claude Code (Claude 3.5 Sonnet) работает лучше и быстрее, чем Codex, но при правильной настройке Codex может работать практически на том же уровне. Codex дешевле и используется как резервный вариант.

## Инструменты и технологии

- **Claude Code** — расширенная версия Claude на основе модели Claude 3.5 Sonnet. Используется как основной е-агент для большинства задач. Доступна по официальной подписке за 17 долларов в месяц. Позволяет интегрироваться с различными системами и выполнять сложные задачи.

- **GPT (ChatGPT)** — модель от OpenAI, используется как альтернативная LLM для е-агентов. Интегрируется с системой для усиления возможностей и проверки работы других моделей.

- **Gemini** — модель от Google, используется для создания е-агентов и как средство для анализа и проверки работы других агентов.

- **Codex** — модель от OpenAI, используется как резервный е-агент на основе GPT. Дешевле, но менее мощный чем Claude Code.

- **Obsidian** — приложение для управления базой знаний и создания связной системы заметок. Используется как внешняя память (третий уровень памяти) для хранения всей информации, истории сессий, базы знаний е-агентов и документации. Позволяет строить графы связей и перелинковки между заметками для быстрого поиска информации.

- **Telegram** — мессенджер, используется как удобный интерфейс для взаимодействия с е-агентами. Позволяет отправлять файлы, голосовые сообщения и получать ответы от агента из любого места через мобильный телефон.

- **n8n** — платформа для автоматизации рабочих процессов. Подключается к е-агенту для автоматического выполнения повторяющихся задач, таких как скрейпинг контента, создание постов, обработка видео и синхронизация данных между системами.

- **Google Workspace (Gmail, Calendar, Google Sheets)** — офисные инструменты, подключаются через CLI от Google для предоставления е-агенту доступа к почте, календарю и таблицам пользователя.

- **FireCrawl** — инструмент для сбора информации и веб-скрейпинга. Позволяет е-агенту собирать данные с различных веб-сайтов и использовать эту информацию для выполнения задач.

- **Perplexity** — поисковая система на основе AI, может быть подключена для расширения возможностей поиска информации в интернете.

- **GitHub** — платформа для управления кодом и версий. Подключается для интеграции со скиллами и хуками, а также для автоматизации разработки.

- **Visual Studio Code** — текстовый редактор, используется для редактирования и просмотра файлов конфигурации е-агентов (Agent.md, Codex Cloud и других).

- **MCP (Model Context Protocol)** — протокол для подключения различных инструментов и сервисов к е-агентам. Используется для интеграции с различными API и системами.

- **CLI инструменты** — командная строка для подключения инструментов Google Workspace и других сервисов к е-агентам. Требует настройки авторизации и конфигурации.

- **Marketplace скиллов и хуков** — онлайн-магазин готовых скиллов и хуков для е-агентов. Содержит множество готовых решений для различных задач.
