## Краткое содержание

Видео демонстрирует запуск Gemma 4 (E2B, 4B параметров) на Raspberry Pi 5 (4 ядра, 8GB RAM) через LM Studio CLI. Автор показывает полный цикл: установка headless LM Studio → загрузка модели на SSD → запуск OpenAI-совместимого API → проброс порта через socat → подключение из Zed Editor на MacBook по локальной сети.

**Ключевые факты:**
- Gemma 4 E2B: 4.5GB, 4B параметров, Apache 2.0 лицензия
- Поддержка: function calling, images/video/audio, 128K контекст
- Скорость генерации на RPi5: ~5-6 минут на развёрнутый ответ (код + объяснения)
- Скорость чтения: на грани комфортного восприятия в реальном времени
- LM Studio CLI: `lms load`, `lms server start --port 4000`
- Проброс: `socat TCP-LISTEN:4001,fork TCP:localhost:4000`

## Применимость к нашей архитектуре

### 1. ClaudeClaw fallback / бюджетный inference

- Gemma 4 E2B с function calling может служить **fallback LLM** для ClaudeClaw при исчерпании API лимитов
- OpenAI-совместимый API = drop-in замена endpoint в grammy bot
- Apache 2.0 = никаких ограничений на коммерческое использование

### 2. Edge processing для n8n

- RPi5 + LM Studio CLI = **локальный LLM-нод** для n8n workflows, не требующих высокой скорости
- Use cases: классификация входящих сообщений, предварительная фильтрация контента, генерация тегов
- Экономия: ≈$0 vs API costs для некритичных задач

### 3. Паттерн: headless LLM server в сети

- `lms server start` + `socat` проброс — минимальная инфраструктура
- Любой клиент с поддержкой custom OpenAI endpoint подключается без изменений
- Zed/VS Code/Cursor + IDE-интеграция через тот же API

### 4. Ограничения (НЕ подходит для)

- Реалтайм чат (5-6 мин на ответ — неприемлемо для пользователей)
- Сложные reasoning-задачи (4B параметров — ceiling быстро)
- Production load (RPi5 на 100% CPU при генерации)
