## Краткое содержание

Видео посвящено оптимизации работы Claude Code путём правильного управления навыками (skills). Автор поделился своим опытом борьбы с чрезмерным расходом токенов и снижением качества работы агента при установке множества скиллов. Видео содержит практические рекомендации по сокращению контекста на 50%, удалению дубликатов скиллов, персонализации навыков и правильной настройке их описаний. Материал предназначен для пользователей Claude Code, которые хотят оптимизировать работу своих AI-агентов без потери функциональности и при этом сэкономить на расходе токенов.

## Ключевые моменты

- **Проблема чрезмерной установки скиллов**: Автор установил 22 скилла из доступных 680+, ожидая повышения производительности, но столкнулся с противоположным эффектом — система стала работать хуже и дороже
- **Первая проблема — расход токенов**: Каждый установленный скилл по умолчанию записывается в файл claude.md, что приводит к значительному расходу токенов при загрузке каждой новой сессии, так как агент загружает всю информацию о скиллах в контекст
- **Вторая проблема — неиспользование скиллов**: Claude самостоятельно принимает решение не использовать установленные скиллы, считая, что может справиться с задачей без них, и тогда скиллы попусту занимают место в контексте
- **Третья проблема — деградация качества**: Когда Claude всё же использует скиллы (особенно для контента типа SEO или копирайтинга), качество ответов снижается до уровня более слабых моделей, так как скиллы часто сгенерированы автоматически или оптимизированы под других пользователей
- **Решение 1 — Lazy Loading режим**: Вместо загрузки полной информации о каждом скилле при старте сессии, в claude.md сохраняется только Description с указанием, когда и в каких случаях использовать скилл
- **Результат Lazy Loading**: Количество строк в claude.md сократилось с 394 до 207 (примерно на 50%), контекст при загрузке сессии сокращен вдвое, при этом функциональность скиллов полностью сохранена
- **Как работает Lazy Loading на практике**: Подробная информация о каждом скилле хранится в отдельном skill.md файле внутри папки скилла и загружается только в момент, когда Claude решает использовать этот конкретный скилл
- **Решение 2 — удаление дубликатов и нерелевантных скиллов**: Автор попросил Claude проанализировать все установленные скиллы на предмет дублирования функциональности и соответствия его текущим задачам
- **Результат анализа дубликатов**: Были выявлены и удалены 7 скиллов, которые выполняли похожие функции (например, два SEO скилла) или были нерелевантны текущему стеку задач, финальное количество скиллов сокращено до 15
- **Решение 3 — персонализация скиллов через Reference File**: Для скиллов, связанных с созданием текстов (копирайтинг, SEO, социальные сети, видеосценарии), создан специальный файл reference.md с описанием голоса бренда, примерами контента, целевой аудитории и правилами написания
- **Применение Reference File**: Когда Claude использует текстовый скилл, он сначала обращается к information из reference.md, чтобы понять, как именно должен писать под конкретный запрос, и только потом применяет скилл с этой информацией
- **Преимущество персонализации**: Скиллы становятся уникальными для конкретного пользователя и его бренда, вместо выдачи общих и шаблонных ответов, характерных для стандартных скиллов
- **Решение 4 — улучшение Description скиллов**: Каждый скилл должен иметь чёткое, детальное описание с указанием причин, когда и почему Claude должен использовать именно этот скилл, а не пытаться решить задачу самостоятельно
- **Как работает правильный Description**: Когда Claude видит задачу и знает из description, что существует специализированный скилл, он понимает, что с этим скиллом справится лучше и автоматически к нему обращается
- **Официальный Skill Creator от Anthropic**: Автор рекомендует установить официальный Skill Creator вместо неофициальных альтернатив, так как он интегрирован глубже во внутренние инструменты Anthropic
- **Возможности официального Skill Creator**: Включает автоматические тесты для проверки эффективности скилла, сравнение работы Claude с скиллом и без него, триггеры для оптимизации момента вызова скилла, и другие продвинутые функции
- **Установка официального Skill Creator**: Можно запросить Claude Code: "Установи мне официальный скилл-креатор от Anthropic" или вручную перейти в раздел скиллов и выбрать плагин Claude Skill Creator
- **Практический результат оптимизации**: После применения всех решений система стала работать быстрее, дешевле (меньше расход токенов), качественнее (скиллы работают корректно и персонализированы) и эффективнее (Claude правильно выбирает скиллы для задач)
- **Долгосрочная стратегия со скиллами**: Не нужно устанавливать все скиллы сразу — лучше начать с необходимых, а новые скиллы добавлять по мере появления специфических задач, которые Claude не может решить самостоятельно

## Инструменты и технологии

- **Claude Code** — основной инструмент, AI-агент от Anthropic, работающий через интеграцию с Telegram. Используется для автоматизации различных задач и работает через систему скиллов (навыков)

- **Skills/Навыки Claude Code** — модульные расширения для Claude, доступные в встроенном магазине (680+ вариантов). Используются для расширения функциональности агента и выполнения специализированных задач

- **Lazy Loading режим** — технический подход к загрузке информации, при котором полные данные о скиллах загружаются только при их использовании, а не при каждом старте сессии

- **Claude.md файл** — основной файл конфигурации Claude Code, содержит промты и метаинформацию о установленных скиллах. Оптимизация этого файла критична для снижения расхода токенов

- **Reference File (reference.md)** — специальный документ, содержащий персонализированную информацию: голос бренда, примеры контента, описание аудитории, правила написания. Используется скиллами для персонализации результатов

- **Skill Creator (официальный от Anthropic)** — инструмент для создания и оптимизации собственных скиллов. Включает автоматические тесты, сравнение производительности, систему триггеров для оптимизации вызова скиллов

- **Telegram** — мессенджер, используемый как интерфейс для взаимодействия с Claude Code (автор построил "мост" между Terminal Claude Code и Telegram для удобства)

- **Gemini/ChatGPT** — использовались автором для верификации проведённых оптимизаций и проверки правильности реализации решений

## К действию

- **Проведите аудит текущих скиллов**: Попросите Claude проанализировать все установленные скиллы на предмет дублирования функциональности и соответствия вашим текущим задачам, составьте список для удаления

- **Реализуйте Lazy Loading**: Переместите полную информацию о скиллах из claude.md в отдельные skill.md файлы в папках скиллов, оставив в claude.md только Description с указанием случаев использования

- **Создайте Reference File**: Подготовьте файл reference.md с описанием вашего голоса бренда, стиля написания, целевой аудитории и примерами контента, подключите его к текстовым скиллам

- **Оптимизируйте Descriptions**: Переработайте Description каждого скилла, добавив чёткие указания о том, когда и почему Claude должен использовать этот скилл вместо решения задачи самостоятельно

- **Установите официальный Skill Creator**: Замените текущий skill creator на официальный от Anthropic, запросив у Claude: "Установи мне официальный скилл-креатор от Anthropic"

- **Создайте документацию изменений**: Используйте Gemini или ChatGPT для верификации проведённых оптимизаций, задокументируйте все внедрённые решения для дальнейшего использования

- **Скачайте готовый отчет**: В описании видео доступен готовый отчет об оптимизации скиллов, который можно загрузить в Claude и попросить реализовать по образцу

- **Интегрируйте Claude Code в Telegram**: Если ещё не сделали, постройте мост между Claude Code в терминале и Telegram для удобного взаимодействия через привычный интерфейс

- **Начните с минимума скиллов**: Удалите все нерелевантные и дублирующиеся скиллы, оставив только необходимые для ваших текущих задач (рекомендуемо 12-15 скиллов)

- **Тестируйте на эффективность**: После оптимизации регулярно проверяйте, использует ли Claude новые скиллы и правильно ли они работают, добавляйте новые скиллы только при необходимости выполнения специфических задач
