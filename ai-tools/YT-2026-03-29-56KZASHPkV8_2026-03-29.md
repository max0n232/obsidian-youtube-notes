## Краткое содержание

Видео посвящено проблеме несоответствия AI-генерируемого контента личному стилю и голосу создателя. Автор Игорь Зугевич делится своей системой обучения AI-агентов (Cloud Code, Claude, Telegram-ботов) для создания контента, который точно соответствует индивидуальному стилю и философии. Основной метод заключается в выгрузке собственного контента в Obsidian, создании графа связей и паттернов для AI-агента, который впоследствии может генерировать новый контент, максимально близкий к оригинальному стилю. Видео предназначено для контент-мейкеров, блогеров, владельцев Telegram-каналов и всех, кто использует AI для создания контента в социальных сетях.

## Ключевые моменты

- **Проблема стандартного подхода**: Обычный способ использования AI для контента (просто написать промпт "напиши мне пост") приводит к созданию шаблонного, низкокачественного контента, который не отражает индивидуальность автора и не набирает просмотры и лайки.

- **Ограничения установки "скиллов"**: Многие надеются, что установка скиллов лучших копирайтеров или профессионалов автоматически заставит AI писать как они, но это не работает и приводит к созданию "мусорного контента".

- **Необходимость голоса бренда**: Для эффективного использования системы нужно иметь четко сформулированный собственный голос и стиль контента. Если этого нет, можно использовать контент другого автора в качестве основы.

- **Система документирования**: Автор создал полную стратегию и систему в трех файлах: "Полная стратегия создания контента на март 2026", "Полная система генерации контента с помощью AI-моделей" и другие документы, которые описывают иерархию работы с контентом.

- **Использование Obsidian как базы знаний**: Obsidian — это инструмент для создания заметок в формате Markdown, который удобен для работы с AI-агентами. В нем создается иерархическая структура контента по разным платформам (Instagram, Telegram, Threads, TikTok).

- **Экспорт контента из Telegram**: Для начала нужно использовать десктопную версию Telegram (отличается от мобильного приложения и имеет больше функций). Через три кнопки на канале выбирается опция "Экспорт истории чата" в формате JSON.

- **Загрузка в AI-агента**: Экспортированный JSON-файл передается AI-агенту (Cloud Code, Claude, Telegram-боту или Codex), который парсит контент и размещает его в Obsidian.

- **Структурирование контента в Obsidian**: AI-агент разбирает весь контент, создавая записи с идентификатором, датой публикации, названием, свойствами и тегами. Каждый пост становится отдельным Markdown-файлом.

- **Создание графа связей**: Агент создает граф (систему связей) в Obsidian, который позволяет по ключевым словам находить релевантный контент. Граф помогает агенту ориентироваться в накопленном контенте.

- **Экономия токенов через контекст**: Благодаря графу в Obsidian AI-агент не тратит большой объем токенов на обработку информации, так как может быстро находить необходимый контент по контексту.

- **Разделение контента по типам**: Весь контент разделяется на текстовый контент и видео-контент, что позволяет агенту лучше понимать, какой тип контента использовать в конкретных ситуациях.

- **Автоматическое обогащение базы**: Когда автор создает новый пост, он может просто закинуть его скриншотом или текстом в AI-агента, и тот автоматически добавит его в базу знаний в Obsidian.

- **Добавление метрик реакций**: Через несколько дней или недель можно добавлять информацию о реакциях на пост (лайки, комментарии, просмотры), что помогает агенту понять, какой тип контента более успешен.

- **Применение паттернов к другим платформам**: После обучения на основе Telegram-контента агент может применять извлеченные паттерны и стиль для создания контента на других платформах (Threads, Instagram, TikTok).

- **Итеративное улучшение**: На начальном этапе первые 5-10 постов могут быть не идеальны, и автор должен вручную корректировать их, давая агенту обратную связь о предпочтениях (например, "меньше английских слов, более человеческий язык").

- **Фиксирование паттернов обратной связью**: Когда автор исправляет пост и объясняет, почему он исправлен, агент сохраняет эти паттерны как постоянные правила для будущего создания контента.

- **Достижение точности после нескольких постов**: Примерно после 5-7, или к десятому посту, AI-агент начинает с высокой точностью понимать стиль, мышление и предпочтения автора и создавать максимально приближенный контент.

- **Автоматизация парсинга**: Агент работает с парсингом и сохранением, то есть может автоматически обрабатывать и организовывать контент без постоянного вмешательства пользователя.

- **Потенциальная автоматизация мониторинга**: В будущем автор планирует поселить агента непосредственно в Telegram-канал, чтобы тот автоматически отслеживал реакции и обновлял базу знаний в реальном времени.

## Инструменты и технологии

- **Obsidian** — приложение для создания взаимосвязанных заметок в формате Markdown; используется как база знаний для хранения и организации контента различных платформ. Позволяет создавать график (граф) связей между заметками для быстрого поиска по ключевым словам.

- **Claude 3.5 Sonnet** — AI-модель от Anthropic; упоминается как одна из современных LLM-моделей для работы с контентом.

- **Telegram Desktop** — десктопная версия Telegram с расширенными функциями для работы с контентом; имеет встроенный функционал экспорта истории чатов в формате JSON.

- **Cloud Code / Claude API** — API для работы с AI-моделью Claude; используется для создания кастомных AI-агентов через интеграции.

- **Telegram-боты (E-Bots)** — специально созданные боты в Telegram, которые интегрируют AI-модели и работают с Obsidian для управления контентом и создания постов.

- **Codex / Code Agents** — специализированные AI-агенты для программной обработки контента и работы с различными форматами данных.

## К действию

1. **Скачайте раздаточные материалы**: Получите три основных файла со стратегией и системой создания контента из первого закрепленного сообщения в Telegram-канале Игоря Зугевича.

2. **Определите свой голос бренда**: Если у вас еще нет четко сформулированного стиля и голоса в своем контенте, начните с написания 10-15 постов самостоятельно перед внедрением AI.

3. **Выгрузите свой контент**: Установите Telegram Desktop, экспортируйте всю историю вашего Telegram-канала в формате JSON (все сообщения, начиная с первого).

4. **Установите Obsidian**: Скачайте Obsidian и создайте структуру папок по платформам (Telegram, Instagram, Threads, TikTok), где будут храниться ваши заметки с контентом.

5. **Загрузите контент в AI-агента**: Передайте экспортированный JSON-файл своему AI-боту (Claude, Cloud Code или Telegram-боту) с инструкцией разобрать контент и загрузить в Obsidian.

6. **Дайте AI инструкции по структурированию**: Попросите агента создать Markdown-файлы для каждого поста с полями: ID контента, дата, название, теги, направление, источник, реакции.

7. **Позвольте создать граф связей**: Дайте AI-агенту команду создать граф (систему связей) в Obsidian для поиска похожего контента по ключевым словам.

8. **Начните с малого объема**: Если весь контент слишком большой, начните с 30-50 постов, чтобы агент быстрее обучился на вашем стиле.

9. **Внедрите итеративное улучшение**: Попросите агента создать 3 варианта нового поста, выбирайте лучший вариант и давайте обратную связь ("меньше английских слов", "более личный тон").

10. **Записывайте паттерны обратной связи**: Каждый раз, когда вы корректируете пост, говорите агенту явно, какой паттерн вы зафиксировали (например, "запиши себе в условиях, что меньше официального языка").

11. **Добавляйте метрики реакций**: Спустя несколько дней после публикации добавляйте информацию о количестве лайков, комментариев, просмотров в метаданные поста в Obsidian.

12. **Проверьте результаты на новой платформе**: После 5-10 пар обучения на Telegram попробуйте создать контент для Threads или Instagram, используя ту же базу знаний.

13. **Планируйте автоматизацию мониторинга**: В будущем рассмотрите возможность встроить AI-агента в Telegram-канал для автоматического отслеживания реакций на посты.

14. **Регулярно обновляйте базу**: Добавляйте новые посты в Obsidian сразу после публикации или еженедельно, чтобы база знаний всегда была актуальной.

15. **Поделитесь системой с командой**: Если у вас есть команда контент-мейкеров, экспортируйте эту систему (файлы из Obsidian) и передайте их своим помощникам для создания последовательного контента.
