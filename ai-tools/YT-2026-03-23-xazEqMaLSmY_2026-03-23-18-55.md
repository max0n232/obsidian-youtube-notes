## Краткое содержание

Видео посвящено анализу нового официального способа подключения Claude Code к Telegram, выпущенного компанией Anthropic, и сравнению его с альтернативным подходом создания полноценных AI-сотрудников через вспомогательных агентов (sub-agents). Автор демонстрирует различие между простым подключением и созданием умной экосистемы агентов с разграниченным доступом к инструментам. Видео предназначено для тех, кто хочет внедрить AI-решения в свой бизнес-процесс и создать полноценные системы автоматизации, которые работают как независимые сотрудники.

## Ключевые моменты

- **Официальный способ подключения от Anthropic**: Новая функция "каналы" позволяет управлять Claude Code через Telegram напрямую, что является ответом на конкурентов вроде OpenCloudBot. Подключение происходит через BotFather в Telegram, где нужно создать бота и получить API-ключ.

- **Проблемы локального использования**: При использовании Claude Code локально через Telegram плагин требуется постоянное включение компьютера, теряются сообщения при перезагрузке, и отсутствует инструментарий для управления зависаниями сессий.

- **Концепция вспомогательных агентов (Sub-agents)**: Вместо простого подключения главного Claude Code к Telegram можно создавать отдельных агентов, которые думают, что они являются главным Claude Code, но при этом подчиняются одному основному агенту-оркестратору.

- **Архитектура системы**: Главный Claude Code размещается на VDS-сервере и не выполняет основную работу, а только мониторит и управляет вспомогательными агентами. Все sub-agents работают через официальную подписку на одном сервере.

- **Неограниченное количество ботов**: Можно создавать множество различных ботов для разных задач (бухгалтерия, копирайтинг, видеоредактирование и т.д.), каждый со своим функционалом и подключением к Telegram.

- **Разграничение доступа**: Каждый sub-agent имеет доступ только к необходимым инструментам и ресурсам. Например, бухгалтеру не нужен доступ к контенту для социальных сетей, копирайтеру нужен доступ к Instagram и Telegram контенту.

- **Доступ к инструментам и интеграциям**: Все sub-agents могут использовать инструменты из основного Claude Code: Obsidian для хранения записей сессий, Notion, n8n для автоматизации, плагины и другие интеграции.

- **Сохранение памяти и состояния**: Каждый агент имеет собственные сессии с сохраненной памятью, что позволяет восстанавливаться после перезагрузок без потери информации и контекста.

- **400-строчный промпт для создания полноценного AI-сотрудника**: Автор разработал расширенный промпт, который преобразует простое подключение к Telegram в полноценного AI-сотрудника с необходимыми инструментами и поведением.

- **Мониторинг и управление через терминал**: Главный Claude Code позволяет управлять вспомогательными агентами через терминал, проверять их статус, видеть, чем они занимаются, и исправлять проблемы (например, слетевшие авторизации).

- **Безопасность и контроль**: Главный агент имеет полный доступ ко всей инфраструктуре и может контролировать, что разрешено и что запрещено делать каждому sub-agent. При нештатных ситуациях главный агент может подключиться и исправить проблему.

- **Функциональность различных ботов**: В системе могут сосуществовать боты для разных задач: бот для адаптации видео, бот для работы с текстовым контентом, бот для взаимодействия через Telegram, боты для социальных медиа.

- **Главный минус официального подключения**: Для того чтобы простое подключение к Telegram стало полноценным AI-агентом, требуется значительная доработка памяти, буфера обмена и интеграций, что теряет основной функционал главного Claude Code.

- **Оркестратор в тени**: Главный Claude Code работает в фоне как невидимый оркестратор, периодически проверяя, что все работает корректно, все авторизации в порядке, и вмешиваясь только при необходимости.

- **Возможность иерархии агентов**: Один из созданных sub-agents может стать главным среди остальных вспомогательных агентов, создавая многоуровневую иерархию системы.

- **Комбинированный подход**: Автор рекомендует использовать официальное подключение Telegram к Claude Code дополнительно, но основной фокус должен быть на создании полноценных AI-сотрудников через систему sub-agents.

## Инструменты и технологии

- **Claude Code (Anthropic)** — основной инструмент для создания AI-агентов, может быть развернут локально, на облачных сервисах или на VDS. Используется как основа для создания вспомогательных агентов.

- **Telegram BotFather** — встроенный инструмент Telegram для создания ботов, получения API-ключей и управления параметрами бота.

- **VDS сервер** — выделенный виртуальный сервер для размещения главного Claude Code и всех вспомогательных агентов. Обеспечивает постоянную доступность системы.

- **Obsidian** — инструмент для сохранения и организации записей сессий AI-агентов, хранит память каждого агента.

- **Notion** — платформа для управления данными и документацией, доступна для интеграции с sub-agents.

- **n8n** — платформа для автоматизации и интеграции различных сервисов, используется для создания рабочих потоков между агентами и внешними системами.

- **GitHub Anthropic** — официальный репозиторий Anthropic, где находится документация по подключению Telegram и список всех плагинов для расширения функциональности Claude Code.

- **Telegram плагин** — официальный плагин от Anthropic для подключения Claude Code к Telegram, находится в разделе external plugins на GitHub.

## Цитаты

> Ребята из Anthropic решили окончательно добить своего главного конкурента OpenCloudBot и выпустили так называемые каналы. Каналы это возможность подключения своего клоуд-кода к телеграмму для управления им через телеграмм.

> Мы же здесь создаем не просто подключение, не просто мост между клоуд-кодом и телеграммом, а мы создаем, грубо говоря, потенциального, либо полноценного сотрудника, у которого есть память, у которого есть доступ к инструментам.

> Каждый из этих сап-агентов думает, что он главный в этом мире, что он думает, что он главный клоуд-код. А вот этот главный клоуд-код, он практически никакую работу не выполняет.

> Я задал своему агенту большую задачу. Он выполнял ее на протяжении 20 минут, потом я не понимал, он завис, либо что-то с ним случилось. Я всегда могу зайти в терминал и спросить, как там дела у этого агента.

> У нас есть главный клоуд-код, который размещен на нашем сервере, мы через главного клоуд-кода создаем себе сап-агента, подключаем его к телеграмму, то есть создаем мост между клоуд-кодом и телеграммой.

> При том, что каждый из этих сап-агентов думает, что он главный, мы можем их ограничивать этими доступами. Например, у бухгалтера, ему не нужно заниматься копирайтингом, поэтому у него нет доступа к нашему контенту.

> Главный минус такого подключения, что для того, чтобы он стал полноценным ИИ-агентом через Телеграм, его нужно очень много до настраивать. До настраивать память, до настраивать буфер обмена.

> Когда же мы работаем непосредственно через сап-агентов, то мы не боимся, что они сделают что-то не то. У них всегда есть главный Клод Код, который ими, грубо говоря, руководит.

> Мой главный клоуд-код заходит, смотрит, что он там делает, и говорит, да, он сейчас работает, поэтому подожди еще, он практически уже закончил.

> Подключить Клод Код к своему Телеграму дополнительно можно, я считаю, что нужно, но все равно создавайте отдельных сап-агентов для того, чтобы работать не просто с Клод Кодом, а для того, чтобы у вас были полноценные AI-сотрудники.

## К действию

- **Развернуть Claude Code на VDS-сервере** вместо локального использования для обеспечения постоянной доступности и отсутствия потерь сообщений при перезагрузках.

- **Создать главного агента-оркестратора**, который будет мониторить и управлять всеми вспомогательными агентами, обеспечивая контроль и безопасность системы.

- **Разработать 400+ строчный промпт** для каждого sub-agent, который превратит простое подключение в полноценного AI-сотрудника с необходимыми инструментами и поведением.

- **Создать иерархию sub-agents** для разных функций в бизнесе (бухгалтер, копирайтер, видеоредактор, менеджер) со своими Telegram ботами и разграниченным доступом.

- **Интегрировать Obsidian** для хранения памяти каждого агента и сохранения контекста между сессиями.

- **Подключить n8n** для создания автоматизированных рабочих потоков между различными агентами и внешними системами.

- **Установить мониторинг через терминал** для отслеживания статуса каждого агента, проверки авторизаций и быстрого решения проблем.

- **Разграничить доступ** каждого агента только к необходимым инструментам и данным в зависимости от их функций.

- **Тестировать работу агентов** на долгие задачи (20+ минут) для выявления потенциальных зависаний и проблем.

- **Документировать все инструкции** для каждого агента и убедиться, что главный Claude Code может их переопределять при необходимости.

- **Создать резервные системы** для управления sub-agents через терминал, чтобы можно было помочь агенту в любой ситуации.

- **Регулярно проверять инфраструктуру** через главного агента, чтобы убедиться, что все авторизации работают, ничего не слетело и все боты функционируют корректно.
