## Краткое содержание

Видео представляет полный гайд по интеграции Claude (Claude AI Code Editor) с n8n для создания специализированного AI-инженера, способного автоматически разрабатывать, тестировать и улучшать n8n workflows. Автор подробно объясняет, почему эта интеграция критически важна для автоматизации процесса автоматизации — то есть для избавления от рутины при создании самих workflows. Видео предназначено для людей без опыта программирования, которые хотят масштабировать свои возможности по созданию автоматизаций через делегирование задач AI. Показывается пошаговая установка Visual Studio Code, подключение Claude расширения, создание специального контекста проекта, установка n8n skills и MCP для Claude, и наконец, создание полноценного workflow через простые текстовые инструкции.

## Ключевые моменты

- **Суть интеграции n8n + Claude**: создание полноценного AI-инженера, который может самостоятельно разрабатывать, администрировать, тестировать и исправлять ошибки в n8n workflows без участия пользователя, только через текстовые инструкции (промпты).

- **Проблема обычных workflows**: при создании большого главного AI-агента в n8n добавление новых инструментов часто вызывает конфликты и ошибки, которые требуют ручного исправления и отладки, отнимающие огромное количество времени.

- **Решение через Claude Code**: вместо ручной отладки пользователь просто даёт текстовую задачу Claude, который автоматически создаёт workflow, встраивает его в существующую систему и исправляет любые возникающие ошибки.

- **Отличие от обычного подключения n8n через MCP**: хотя можно подключить n8n к Claude через Model Context Protocol, обычный Claude создаст workflow без глубокого логического анализа; специализированный Claude с n8n skills и базой знаний будет сначала логически спланировать архитектуру, а затем реализовывать.

- **Режим Plan Mode в Claude**: это режим, где Claude сначала анализирует задачу, строит логическую схему взаимодействия компонентов, а потом реализует код; это аналог Extended Thinking в Claude и помогает избежать ошибок в future workflows.

- **Установка Visual Studio Code**: бесплатное приложение Visual Studio Code Open Source AI Edition скачивается с официального сайта и служит базой для работы с Claude расширением.

- **Установка Claude расширения**: официальное расширение Claude от Anthropic устанавливается из маркетплейса VS Code (более 3 миллионов установок), требует авторизации через Cloud.

- **Важная настройка в Claude**: необходимо активировать галочку, которая даёт Claude разрешение на автоматическое выполнение операций без бесконечных подтверждений пользователя, иначе процесс сильно замедляется.

- **Создание проекта**: в VS Code создаётся новая папка (например, "n8n-test"), которая становит рабочей директорией для всех workflows и контекстных файлов.

- **Cloud.md контекстный файл**: создаётся файл Cloud.md (Markdown), который будет служить контекстной базой для Claude, описывая правила, логику и знания о n8n.

- **Инициальный промпт для контекста**: первый промпт объясняет Claude, что он будет создавать workflows в n8n, использовать n8n skills и n8n MCP, и что он должен выполнять это через простые инструкции пользователя.

- **Claude создаёт папки и структуру**: при запуске режима Plan Mode с правильным промптом Claude самостоятельно создаёт необходимую структуру папок (code, docs, workflows) и файлы (README, mcp.json и т.д.).

- **Установка n8n skills и MCP**: второй этап включает установку двух ключевых компонентов — n8n Skills (база знаний о каждой ноде и workflow паттернах) и n8n MCP (протокол подключения), которые получаются из github репозиториев.

- **Получение API-ключа n8n**: в настройках n8n (Settings → n8n-api) создаётся API-ключ, который вводится в файл mcp.json для авторизации Claude на доступ к аккаунту n8n.

- **Ввод URL n8n**: в mcp.json также вводится прямая ссылка на инстанс n8n (без конечного слэша), чтобы Claude мог подключаться и создавать workflows напрямую.

- **Проверка подключения**: после настройки можно попросить Claude просто список всех существующих workflows в n8n, чтобы убедиться, что подключение работает корректно.

- **Создание workflow через Claude**: после полной настройки пользователь просто пишет задачу на русском или английском, включает режим Plan Mode, и Claude сам создаёт полноценный workflow из нескольких нод, загружает его в n8n и предоставляет инструкции по использованию.

- **Время выполнения**: создание контекста проекта занимает 4 минуты, установка инструментов — 1 минута (при повторном запуске, первый раз 20 минут), создание workflow — 2 минуты с подробным объяснением каждого шага.

- **Использование токенов**: при работе с Claude на русском языке расходуется больше токенов, поэтому рекомендуется писать промпты на английском для оптимизации использования квоты.

- **Дальнейшие возможности**: после настройки Claude можно не только создавать новые workflows, но и редактировать существующие, добавлять AI-агентов, улучшать и адаптировать шаблоны n8n под конкретные нужды буквально в несколько минут.

## Инструменты и технологии

- **Visual Studio Code** (https://code.visualstudio.com/) — бесплатный open-source редактор кода с поддержкой AI расширений, используется как базовая платформа для интеграции Claude.

- **Claude Code Editor расширение** — официальное расширение от Anthropic для VS Code, добавляет возможность взаимодействия с Claude AI прямо в редакторе; требует авторизации через Cloud.

- **n8n Skills** (github репозиторий) — специализированная база знаний о каждой ноде n8n, workflow паттернах и best practices, загружается в Claude для создания максимально оптимальных workflows.

- **n8n MCP (Model Context Protocol)** — протокол подключения n8n к Claude, позволяет Claude напрямую создавать, редактировать и тестировать workflows в инстансе n8n пользователя.

- **n8n** (https://n8n.io/) — платформа no-code для создания workflows и автоматизаций, используется как основной инструмент выполнения, Claude создаёт workflows в n8n.

- **Google Translator** (браузер) — встроенный переводчик для браузера, используется для перевода промптов и ответов Claude с английского на русский и обратно для оптимизации работы.

- **mcp.json файл** — конфигурационный файл, содержит API-ключ и URL инстанса n8n, необходим для авторизации Claude при подключении к n8n.

- **Cloud.md** — файл контекста проекта на Markdown, содержит инструкции, логику и knowledge base для Claude по работе с n8n, используется как долгосрочная память AI.

## К действию

- **Установить Visual Studio Code** и официальное расширение Claude Code от Anthropic для создания рабочего окружения.

- **Создать структуру проекта** в VS Code (папка для n8n workflows) и инициализировать Cloud.md файл как контекстную базу.

- **Получить API-ключ в n8n** (Settings → n8n-api → create-api-key) и скопировать URL инстанса n8n.

- **Настроить mcp.json файл** с API-ключом и URL n8n для подключения Claude к вашему инстансу.

- **Активировать режим Plan Mode** в Claude для логического планирования перед реализацией любых workflows.

- **Активировать разрешение на автоматическое выполнение операций** в настройках Claude, чтобы избежать постоянных запросов подтверждения.

- **Установить n8n Skills и n8n MCP** через соответствующие github репозитории для расширения знаний Claude о n8n.

- **Проверить подключение**, попросив Claude вывести список всех workflows в вашем n8n инстансе.

- **Написать первый промпт** для создания простого workflow (например, обработка текста через AI и возврат результатов) и позволить Claude создать его автоматически.

- **Использовать английский язык** для промптов Claude, чтобы оптимизировать расход токенов и получить более быстрые результаты.

- **Экспериментировать с редактированием существующих workflows** — просто приложите ссылку на workflow и попросите Claude его улучшить или добавить новые ноды.

- **Сохранять контекст между сессиями** через Cloud.md файл, чтобы Claude запоминал специфику вашего проекта и мог продолжать работу в будущем.

- **Документировать все workflows**, которые создаёт Claude, чтобы иметь записи всех инструкций и логики для будущей поддержки.
