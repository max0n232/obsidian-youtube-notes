## Краткое содержание

Видео демонстрирует полный процесс интеграции n8n с Claude Code в Visual Studio Code для автоматизации создания и управления рабочими процессами без знания программирования. Автор показывает, как настроить специализированного AI-инженера, который может самостоятельно проектировать, создавать, редактировать и отслеживать ошибки в n8n workflows. Это позволяет делегировать рутинную работу по администрированию автоматизаций искусственному интеллекту. Видео адресовано людям, которые хотят масштабировать свою работу с n8n, не являясь программистами, и готовы использовать современные AI-инструменты для упрощения процесса разработки.

## Ключевые моменты

- **Основная цель интеграции**: Создание AI-инженера на базе Claude Code, который может полностью управлять жизненным циклом n8n workflows без участия человека в техническихдеталях

- **Проблема без Claude Code**: При добавлении новых инструментов к главному AI-агенту возникают ошибки, которые требуют ручного исправления и отладки, что отнимает много времени и сил

- **Решение через Claude Code**: Вместо ручной отладки можно делегировать всю работу Claude, который будет создавать workflows, внедрять их, исправлять ошибки и администрировать процессы автоматически

- **Двухуровневая автоматизация**: Если раньше автоматизация убирала рутину из бизнес-процессов, то теперь Claude убирает рутину из самого процесса создания и управления автоматизациями

- **Сценарий с переиспользованием workflows**: Если вы заказали workflow у разработчика, но не можете найти его позже для обновления, Claude Code может самостоятельно обновить, улучшить и модернизировать этот workflow

- **Преимущество перед ChatGPT**: Claude Code может создать специализированного агента с глубокими знаниями n8n, в отличие от обычного ChatGPT, который создаёт workflow без предварительного логического планирования

- **Режим Planning Mode**: Это критический режим, при котором Claude сначала строит логическую структуру workflow, анализирует все зависимости, а затем начинает программировать, что значительно снижает количество ошибок

- **Инструменты для Claude**: Необходимо установить n8n Skills (база знаний по n8n ноды и документация) и n8n MCP (специальный протокол для подключения Claude к n8n API)

- **Установка Visual Studio Code**: Требуется скачать бесплатное приложение Visual Studio Code (Open Source AI Code Edition) и установить расширение Claude Code от Anthropic

- **Создание проекта**: Внутри VS Code создаётся новая папка проекта, где будут храниться все файлы конфигурации, документация и логика для работы Claude с n8n

- **Первый промпт - контекст проекта**: Необходимо создать файл ClaudeMD с промптом, который объясняет Claude, что нужно помочь в создании системы для построения и управления n8n workflows

- **Второй промпт - установка инструментов**: После создания контекста используется специальный промпт для загрузки в Claude знаний о n8n из GitHub репозиториев (n8n skills и n8n MCP)

- **Конфигурация mcp.json**: В файле mcp.json нужно добавить API ключ n8n и URL инстанса n8n для прямого подключения Claude к вашему аккаунту

- **Получение API ключа**: В настройках n8n создаётся новый API ключ с названием mcp-n8n, который затем вводится в конфигурацию Claude Code

- **Тестирование подключения**: После конфигурации можно попросить Claude проверить подключение, запросив список всех workflows из вашего n8n аккаунта

- **Различные режимы Claude Code**: Ask Before Edits (спрашивает перед изменениями), Edit Automatically (автопилот), Planning Mode (сначала планирует, потом делает), Bypass Permissions (самый опасный, обходит все проверки)

- **Процесс создания workflow**: После настройки Claude можно попросить создать новый workflow в режиме Planning Mode, указав требования на русском языке, и Claude самостоятельно спроектирует и реализует его

- **Экономия токенов**: Общение с Claude на английском языке требует значительно меньше токенов, чем на русском, поэтому рекомендуется использовать Google Translate для оптимизации

- **Скорость работы**: Базовая настройка подключения занимает 4-20 минут в зависимости от наличия зависимостей, создание workflow занимает 2-10 минут в режиме Planning Mode

- **Дальнейшие применения**: Claude Code может не только создавать и редактировать workflows, но и разрабатывать фронтенд приложения, сайты и другие инструменты поверх n8n

## Инструменты и технологии

**Visual Studio Code (Open Source AI Code Edition)**
- Ссылка: https://code.visualstudio.com/
- Описание: Бесплатный редактор кода с поддержкой расширений и встроенной интеграцией AI
- Использование: Основная платформа для работы с Claude Code и управления n8n workflows

**Claude Code (расширение Anthropic)**
- Официальное расширение от Anthropic для Visual Studio Code
- Описание: AI-агент, встроенный в редактор, может программировать, проектировать и управлять кодом
- Использование: Создание, редактирование и управление n8n workflows через естественный язык

**n8n API**
- Описание: REST API сервиса n8n для программного управления workflows, ноями и другими элементами
- Использование: Claude Code использует API для создания и модификации workflows в n8n

**n8n Skills (GitHub репозиторий)**
- Ссылка на GitHub: Упомянуто в видео как репозиторий с документацией и примерами n8n
- Описание: База знаний со всеми доступными ноями n8n, их параметрами и примерами использования
- Использование: Загружается в Claude для специализации в создании n8n workflows

**n8n MCP (Model Context Protocol)**
- Описание: Специальный протокол для подключения Claude к n8n API
- Использование: Позволяет Claude напрямую взаимодействовать с n8n для создания и управления workflows

**mcp.json (конфигурационный файл)**
- Описание: Файл конфигурации для хранения API ключа n8n и URL инстанса
- Использование: Содержит учётные данные для подключения Claude к вашему n8n аккаунту

**Google Translate (браузер или мобильное приложение)**
- Описание: Инструмент для перевода промптов с русского на английский и обратно
- Использование: Экономия токенов при работе с Claude (английский требует меньше токенов)

**GitHub**
- Описание: Платформа для хранения открытых репозиториев с n8n skills и документацией
- Использование: Источник специализированных знаний для обучения Claude

## К действию

- **Шаг 1: Установка окружения** - Скачайте Visual Studio Code с сайта code.visualstudio.com и установите расширение Claude Code от Anthropic, авторизуйтесь в нём через Claude API

- **Шаг 2: Создание проекта** - Создайте новую папку проекта в VS Code и инициализируйте её как проект n8n, в которой будут храниться все конфигурации и документация

- **Шаг 3: Создание контекста** - Скопируйте первый промпт из ресурсов (доступен в Telegram канале) и добавьте его в VS Code, запустите режим Planning Mode для создания структуры проекта

- **Шаг 4: Загрузка n8n знаний** - Используйте второй промпт для загрузки n8n Skills и n8n MCP из GitHub репозиториев, это даст Claude специализированные знания о n8n

- **Шаг 5: Получение API ключа** - В своём n8n аккаунте перейдите в Settings → n8n API → Create API Key и назовите его mcp-n8n, скопируйте полученный ключ

- **Шаг 6: Конфигурация подключения** - В файле mcp.json добавьте скопированный API ключ в поле n8n-api-key и URL вашего n8n инстанса (без слэша в конце) в поле url

- **Шаг 7: Тестирование подключения** - Попросите Claude проверить подключение, запросив список всех workflows из вашего n8n, если получите результат - всё настроено правильно

- **Шаг 8: Создание первого workflow** - Используйте промпт на русском языке для описания нужного вам workflow, выберите режим Planning Mode и дайте Claude создать workflow в вашем n8n

- **Шаг 9: Итерация и улучшение** - Если workflow требует исправлений, попросите Claude отредактировать его, добавить новые ноды или изменить логику - всё это займёт несколько минут

- **Шаг 10: Масштабирование** - Начните выгружать свои существующие workflows в Claude, просите его улучшать, обновлять и модернизировать их, постепенно переходя на полностью автоматизированное управление

- **Оптимизация токенов** - По возможности общайтесь с Claude на английском языке для экономии токенов, используйте Google Translate для перевода длинных промптов

- **Создание специализированного агента** - После базовой настройки создайте специального AI-инженера путём добавления дополнительного контекста с вашими требованиями и стандартами разработки

- **Документирование workflow** - Просите Claude создавать подробную документацию для каждого workflow, включая инструкции по использованию и управлению

- **Автоматизация администрирования** - Делегируйте Claude все задачи по администрированию workflows - мониторинг ошибок, обновление, добавление новых инструментов к главному AI-агенту
